> 본 dictionary 는 한 명의 KR 시장 fnguide DataGuide 일괄 export 31개 CSV (`E:\personal_bot_kr_data_fnguide_backup\data_fnguide`, 사적 자산, 비공개) 의 **실제 row 검사 evidence** 기반.
> 학생 본인의 fnguide 라이선스로 받는 데이터는 query 범위에 따라 다를 수 있다. 본 doc 은 일반적 schema convention 만 명시.

## 1. Export 일반 schema

확인 범위: 31개 일괄 CSV header (row 0-15) + 각 파일 첫 ticker (`A005930` 삼성전자, 한 case 는 `A252500` 세화피앤씨) row.

- **Encoding**: cp949 (한글 Windows 코드페이지). UTF-8 아님. Python `open(p, encoding='cp949')` 또는 pandas `read_csv(..., encoding='cp949')`.
- **Layout**: long-form. 행 = (ticker, item) 조합. 열 = 6 메타데이터 + ~9349 일자 컬럼.
- **Date 범위**: 대부분 `1990-01-01 ~ 2025-10-24 ~ 2025-10-31` (export 시점 기준). 일자 컬럼 ~9345-9390개.
- **Universe 폭**: 첫 ticker 가 거의 모두 `A005930` 삼성전자 (확정 31파일 중 30) — 즉 KOSPI/KOSDAQ 상장 전체 ticker 가 ticker-block 으로 누적 (한 ticker 의 모든 item 끝나면 다음 ticker 의 item 들 이어짐).

### 1.1 Header layout (모든 파일 공통)

| row | col 0 | col 1 | col 2 | col 3+ |
|---|---|---|---|---|
| 0 | `Calendar Basis` | | | |
| 1 | `Portfolio` | `전부` | `Default` / `원` | |
| 2 | `Item` | | | |
| 3 | `Frequency` | `일간` | `KRW` | |
| 4 | `Non-Trading Day` | `Previous` | `Asc` | |
| 5 | `Include Weekend` | `NONE` | | |
| 6 | `Term` | `19900101` | `Current(20251030)` | (export 시점) |
| 7 | `Symbol` | `Symbol Name` | `Kind` | `Item`, `Item Name`, `Frequency`, `1990-01-01`, `1990-01-02`, ... |
| 8+ | `A005930` | `삼성전자` | `SSC` | `S410000650`, `수정시가(원)`, `DAILY`, `447`, ... |

- `Kind` 예: `SSC` (Securities/Stock).
- `Item` = item code (fnguide 표준), `Item Name` = 한글 정의.
- 컬럼 의미: 데이터 값은 한 ticker × item 의 day-by-day time series.

### 1.2 학생용 read 패턴 (Python pandas)

```python
import pandas as pd
df = pd.read_csv(
    "한국주식_가격과변동성.CSV",
    encoding="cp949",
    skiprows=7,     # row 0-6 metadata skip
    header=0,       # row 7 = column header (Symbol, Symbol Name, Kind, Item, Item Name, Frequency, dates...)
    low_memory=False,
)
# long form (ticker, item) rows; columns dates
# wide → long melt:
id_cols = ["Symbol", "Symbol Name", "Kind", "Item", "Item Name", "Frequency"]
long = df.melt(id_vars=id_cols, var_name="date", value_name="value")
long["date"] = pd.to_datetime(long["date"])
```

### 2.1 Suffix convention

- `(E3)` = 3-month consensus
- `(E3, 중간)` = consensus median
- `(E3, 최대)` / `(E3, 최소)` = max/min
- `(E3, 1W Chg)` / `(1M Chg)` / `(3M Chg)` / `(6M Chg)` = N-week/month change of consensus
- `(E3, QoQ)` = quarter-over-quarter
- `(직전4분기)` = trailing 4 quarters (TTM)
- `(연율화)` = annualized
- `(3년평균)` = 3-year average
- `(평균)` = period average
- `(잠정)` / `(확정실적속보)` = preliminary / confirmed-flash
- `(현금배당반영)` = adjusted for cash dividend
- `(보통+우선)` = common + preferred (combined)
- `(만원)` / `(억원)` / `(원)` / `(%)` / `(원,US센트)` — 단위

## 4. KAIST FE 학생 사용 시 주의사항

### 4.1 라이선스 확인

- **KAIST 학생**: fnguide DataGuide + **CheckExpert** 양쪽 모두 KAIST 라이선스 available (confirmed). `docs/checkexpert-howto.md` 참조.
- bulk CSV export 권한이 없는 라이선스 (web query 만) 가 대부분 → 본 dictionary 의 item code 가 web query 에서도 그대로 사용 가능.
- DataGuide Python API (`pydataguide` 류) 가 있는 학교도 있고 없는 학교도 있음.
- 다른 학교 (서울대 / 연세대 / KU / Sogang MFE 등) 학생: 본교 라이선스 확인 의무 — KAIST 외에는 fnguide / CheckExpert 라이선스 보장 없음.
- agent 가 데이터 다운로드 자동 수행 금지 — 학생 task 로 escalation (AGENTS.md §6.1 양식).

### 4.2 회계기준 변경 (timing)

- 한국 IFRS 의무 도입: 2011 회계연도부터 상장 대기업 (단계적). 2013 부터 모든 상장사 K-IFRS.
- 2011 이전 데이터는 K-GAAP 기반 → IFRS 정의와 약간 다른 항목 존재 (영업이익 정의, 영업외손익 분류 등).
- 6000* 재무제표 계정의 1990~2010 구간 사용 시 IFRS-K-GAAP 일관성 보장 안 됨 명시 의무.

### 4.3 Universe 변동

- KOSPI ~800 / KOSDAQ ~1600 (현재). 1990 년대는 미상장 종목 대부분 → 결측 정상.
- 우선주 (`-P` 접미사 — backup 에는 ticker prefix `A` 뒤 7자리 숫자, 우선주는 별도 ticker code) / ETF / SPAC / 리츠 분리 처리.
- 상장폐지 종목은 backup snapshot 의 export 시점 기준 universe 에서 제외돼 있을 가능성 — survivorship bias 주의.

### 4.4 컨센서스 (E3) 의미

- `E3` suffix 가 fnguide convention 으로 **forward 3-month rolling consensus** 의 의미인지, **3개 brokerage 평균** 인지, 다른 정의인지 fnguide 매뉴얼 확인 의무. paper 의 consensus 정의와 정확히 일치하는지 검증 후 사용.
- `(중간)` = median, `(최대)`·`(최소)` = analyst dispersion 으로 사용 가능.
- `1W Chg / 1M Chg / 3M Chg / 6M Chg` = revision factor 로 사용 가능 (분자: 현재 consensus, 분모: N 전 consensus).

### 4.5 수급 (CI20*) 정의

- 단위 `(만원)` 과 `(원)` 혼재 → 파일별 단위 확인 의무.
- 투자자 분류 type 정의 = KRX 공식 분류 + fnguide 가공.
- `연기금등` / `연기금` 미묘한 차이 (등 포함 여부) — fnguide 매뉴얼 확인.
- `등록외국인` (LP 등록) vs `기타외국인` 구분.

### 4.6 자사주 / 우선주 처리

- 시가총액 (`S410001200`) 정의: ticker 자체 시가 × (보통 + 우선) ? 본 backup 에서는 `시가총액 (티커-상장예정주식수 포함)(원)` 로 표기 → 우선주 시총 별도 ticker 로 따로 발행 가능성 확인 의무.
- 외국인보유비중 = 보통+우선 결합.

## 5. 학생이 해야 할 일 (User task escalation 양식)

agent 가 데이터 필요 만나면 본 dictionary 참조해서 다음 양식으로 `thesis_state/03_user_tasks.md` 에 append (자세한 규약은 AGENTS.md §5.3):

```
## Task U-N: <한 줄: fnguide DataGuide 에서 <variable> 다운로드>
- **왜 필요한가**: <paper §X 의 <원저 변수>. fnguide item code <Fxxx 또는 Sxxx 또는 6000xxxxxx>.>
- **무엇을 해야 하나**:
  1. KAIST 도서관 fnguide 라이선스 로그인
  2. DataGuide → 종목 선택 → 전체 KR 상장 (KOSPI + KOSDAQ)
  3. Item 검색 → "<item name>" 또는 직접 code "<Fxxx>" 입력
  4. Term: <YYYYMMDD ~ YYYYMMDD>
  5. Frequency: <DAILY / QUARTERLY / ANNUAL>
  6. Export → CSV → `data/raw/<filename>.csv` 저장 (encoding cp949)
- **완료 검증**: `data/raw/<filename>.csv` 존재 + 첫 행에 "Calendar Basis" 헤더 + 컬럼수 > 9000.
- **막혔을 때 대안**:
  - 라이선스 미보유 → 지도교수·연구실 선배 통해 다른 학생 라이선스 빌리기 (학교 정책 확인)
  - item code 미식별 → 본 dictionary §2 prefix family 표 재확인, 또는 fnguide 고객 지원
  - CSV 너무 큼 → universe 축소 (KOSPI 만 또는 KOSPI200 만)
- **상태**: blocked
```
